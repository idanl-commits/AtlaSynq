Security posture

Every control your security team will ask about — answered before they ask.

SOC 2 Type II

Annual third-party audit covering availability, confidentiality, and security controls.

SSO / SAML

Okta, Azure AD, and any SAML 2.0 provider. MFA enforced at the identity layer.

Full audit log

Every query, approval decision, and admin action is logged with user identity and timestamp.

RBAC at connector layer

Role policies are enforced per-query. A Finance agent cannot reach Sales data, even with a crafted prompt.

Human-in-the-loop approvals

Write actions are queued for explicit human approval before any system is modified.

Zero training on customer data

Your queries and documents are never used to train foundation models. Data is processed and discarded.

Encryption at rest & in transit

AES-256 at rest. TLS 1.2+ in transit. Keys managed in a dedicated KMS with rotation policies.

VPC / on-prem (Enterprise)

Private deployment in your cloud VPC or on-premises for air-gapped and regulated environments.

How permissions work

Permissions are not set-and-forget — they are evaluated on every single query.

Step 01

Role policies

Admins define which connectors, data scopes, and write actions each role is permitted to use. Changes apply immediately without redeployment.

Step 02

Query-time enforcement

When a user sends a query, the agent checks the role policy before touching any connector. Unauthorized paths return a permission denial, not a data leak.

Step 03

Citations + approvals

Read answers cite their source records. Write actions are placed in a review queue. Every step is logged with identity, timestamp, and outcome.
